This crawler can (hopefully) help you speed up converting existing MDN browser compatibility data to the new browser-compat-data format.

  1. Append the name of a web API to https://mdncrawler.glitch.me/crawl/. E.g. if you want to crawl https://developer.mozilla.org/en-US/docs/Web/API/Metadata you enter https://mdncrawler.glitch.me/crawl/Metadata.
  2. Wait a while... the crawler may have to visit a lot of pages. Sometimes it times out if you visit a page with a lot of links. You can try running the site locally by downloading the Git repo: https://github.com/kaycebasques/mdncrawler
  3. You should eventually get back formatted data which is pretty close to the browser-compat-data format.

Report bugs to https://twitter.com/kaycebasques or kayce@google.com or https://github.com/kaycebasques/mdncrawler/issues/new. There's probably more that I can automate, so you are welcome to send me suggestions.
